# Transcript → 公众号文章生成规范

## 核心原则

**重新创作，而非整理笔记。** Transcript 是原材料，目标是创作一篇独立成文的高质量内容。口语化、重复、填充词全部丢弃，专注提取和重组有价值的洞见。

## 内容价值层次

按优先级提取信息：

1. **独家洞见** - 讲者的原创观点、独特经验、反常识结论
2. **实用知识** - 具体方法、工具推荐、可操作的建议
3. **案例故事** - 真实案例、具体数据、生动例子
4. **行业趋势** - 前瞻性观点、市场分析、发展预测
5. **思维框架** - 思考模式、分析方法、决策逻辑

**原则：一个洞见胜过十个常识。**

## 文章结构设计

### 灵活框架（根据内容调整）

**Option A: 洞见驱动型**（适合观点类内容）
```
金句开场 → 核心洞见阐述 → 多角度论证 → 实践应用 → 思考启发
```

**Option B: 方法指导型**（适合教程类内容）
```
问题场景 → 解决方法 → 步骤拆解 → 案例验证 → 行动建议
```

**Option C: 故事叙述型**（适合经验分享）
```
故事背景 → 关键转折 → 深层思考 → 普遍规律 → 个人启示
```

### 标准结构模板

```
[frontmatter]
---
title: "选定的爆款标题"
author: ""
summary: "120字内精准摘要，突出独特价值"
tags: ["相关标签"]
---

[开场：强力钩子，80-150字]
- 反常识结论 OR 具体场景 OR 尖锐问题
- 0.5秒抓住注意力，承诺明确价值

[核心部分：2-4个板块，按逻辑展开]
- 每板块围绕一个主要观点
- 观点 + 解释 + 例子 + 小结
- 板块间有逻辑递进关系

[收尾：升华+行动，80-120字]
- 总结核心价值，但不重复正文
- 给出具体的下一步行动
- 留下值得思考的问题
```

## 字数与节奏控制

- **总字数：1200-1800字**（优质长文更有影响力）
- **开场：80-150字**（快速抓住注意力）
- **核心部分：900-1400字**（深度展开，但不拖沓）
- **收尾：80-120字**（升华总结，不重复）

## 默认阅读体验准则

以下三条与务实风格同级，是所有文章的硬性要求：

### 有条理
- 每篇文章有清晰的主线：一句话说得清楚"这篇文章讲什么"
- 段落之间有逻辑承接，不能跳跃；用"因此""但是""换句话说"等词显式连接
- 每个自然段只表达**一个**核心意思，超过一个则拆段
- 小标题准确概括该节内容，不用模糊标题如"延伸思考"

### 看起来舒服
- 正文段落≤100字，超出则拆分
- 连续3个以上并列要点必须用列表，不堆在一段里
- 每300-500字设一个小标题，避免长篇无标题文字墙
- 重要结论或关键数字用**加粗**，但每篇加粗不超过5处

### 通俗易懂
- 专业术语首次出现必须用括号或破折号给出白话解释
- 复杂概念用类比：先说"它像什么"，再解释"哪里不同"
- 句子平均长度≤20字；遇到长从句，拆成两句
- 不假设读者有背景知识：每个论点自给自足，不要求读者"你懂的"

## 默认写作风格：务实

**务实风格**是本 skill 的默认基调，优先于以下所有风格技巧：

- **直接给结论**：不铺垫，不卖关子，核心观点放在最前面
- **可操作性第一**：每个观点都要有"怎么做"，不停留在"是什么"
- **去掉情绪煽动**：不用"震惊""颠覆""必看"等词；用事实说话
- **避免虚华表达**：不写"在当今社会""我们不得不思考"等空话
- **实例优于原则**：有具体案例、数据、步骤时，省略抽象总结

## 写作风格升级

### 语言特色
- **对话感强**：像资深朋友在分享见解，不是专家在讲课
- **具体胜过抽象**：用具体的数字、名字、场景替代模糊表述
- **短句为王**：平均句长15-25字，避免超过35字的复杂句
- **情绪传导**：适当使用感叹、疑问来传递作者的情绪和态度（务实风格下克制使用）

### 高级技巧
1. **金句穿插**：每300-400字安排一个memorable的表述（务实风格下确保金句有实质内容，非口号）
2. **对比制造张力**：新旧观念对比、预期vs现实对比
3. **数据说话**：具体数字比"很多"、"大幅"更有说服力
4. **读者代入**：用"你"而非"我们"，用"想象一下"引导思考

## 写作指导的应用

如果用户提供了写作指导，**优先级高于以上默认规范**：

| 指导类型 | 应用方式 |
|----------|----------|
| 目标受众 | 调整语言深度：小白用户多解释，专业用户更简洁 |
| 文章风格 | 参考 `writing-patterns.md` 选择对应模式 |
| 重点内容 | 重点部分用更多篇幅，采用多角度论证 |
| 补充背景 | 融入用户提供的行业信息或个人经验 |

**参考资源**：优先查阅 `references/writing-patterns.md` 获取具体的写作模式和表达技巧。

## 爆款标题生成策略

生成8个候选标题，覆盖更多风格和角度。

**硬约束：标题不超过 64 字节（约 20 个中文字符），这是微信草稿 API 的限制。**

标题走**短、硬、可记**的题眼式写法。优先保留一个核心张力词，让用户一眼记住。

### 核心类型（必选4个）
1. **价值直给型**：`[主题][具体收获]`
2. **认知颠覆型**：`[常见认知]错了`
3. **场景代入型**：`[具体场景]怎么办`
4. **方法指导型**：`如何[达成目标]`

### 进阶类型（从中选4个）
5. **对比反差型**：`[A]，不是[B]`
6. **时间紧迫型**：`[时间期限]练成[目标]`
7. **内部视角型**：`[内行]才懂的[真相]`
8. **数据驱动型**：`[具体数据]背后`
9. **趋势洞察型**：`[现象]正在改变你`
10. **个人成长型**：`[经历]教会我什么`

### 标题优化原则
- **≤20 中文字符 / 64 字节**（微信 API 硬约束）
- **短、硬、可记**（优先于解释完整）
- **关键词前置**（方便搜索）
- **承诺要兑现**（标题党必死）
- **少用虚词**（避免“的、一个、如何做到”之类拖长节奏的写法）
- **优先保留张力词**（如紧张、边界、松弛、课题、表达）

```yaml
---
title: ""        # 由用户在 Step 4 选定后填入

value_type: ""   # 价值类型：洞见/方法/案例/趋势/思维框架

```

## 执行检查单

生成文章后自检：

**内容维度**
- [ ] 是否有至少1个独特洞见（区别于常识）？
- [ ] 每个主要观点都有具体例子支撑？
- [ ] 读完后读者能获得可操作的建议？

**表达维度**
- [ ] 开场3秒内抓住注意力？
- [ ] 平均句长控制在25字以内？
- [ ] 专业词汇有解释或上下文帮助理解？

**价值维度**
- [ ] 文章解决了读者的实际问题？
- [ ] 提供了不容易获得的信息？
- [ ] 读完有"原来如此"的顿悟感？

通过7/9项 → 优秀；5-6项 → 良好；<5项 → 需要重写核心部分。
