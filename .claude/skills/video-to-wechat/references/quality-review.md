# 公众号文章质量审核标准

## 审核流程

作为 subagent，读取文章和原始 transcript，按以下标准逐项检查，**直接修改文章**，最后输出简短审核摘要。

---

## 1. 结构完整性检查

| 项目 | 标准 | 自动修复 |
|------|------|----------|
| 开头 | 有吸引力的切入点（场景/问题/反常识），100-200字 | 重写开头 |
| 小节数量 | 3-4 个正文小节，每节有 H2 标题 | 拆分或合并 |
| 结尾 | 有总结+行动引导，100-150字 | 补充结尾 |
| frontmatter | 含 title/author/summary | 补充缺失字段 |

**如结构不完整，直接补充修复。**

---

## 2. 信息密度评分

评分维度（每项 0-10 分）：

- **核心论点清晰度**：读者能否在 30 秒内从标题+摘要明白文章价值
- **例子具体性**：每个论点是否有具体例子/数据支撑，而非泛泛而谈
- **冗余度**（越低越好）：是否有重复表达同一观点的段落

**信息密度低的常见问题及修复**：
- 同一个意思说了两遍 → 合并为一段
- 连接词过多（"另外"、"此外"、"同时"密集出现）→ 删除冗余连接
- 结论段重复正文内容 → 结尾只做升华，不做重复

总分 < 20 → 需要大改；20-25 → 小修；> 25 → 通过

---

## 3. 信息保真度检测

对照原始 transcript 检查：

**必须保留**：
- 讲者的核心论点（即使改写了表述）
- 具体数字、案例名称、时间节点
- 讲者的原创观点或结论

**允许改变**：
- 表述方式（口语 → 书面）
- 信息顺序（按逻辑重组）
- 细节详略（删减次要内容）

**不允许**：
- 添加 transcript 中没有的事实（不捏造）
- 反转讲者观点
- 删除核心论点

如发现保真度问题，**标注出来并询问用户**（而非自动修改，因为这可能是主观判断）。

---

## 4. 公众号可读性要求

| 要求 | 标准 |
|------|------|
| 段落长度 | 单段不超过 150 字，否则拆分 |
| 句子长度 | 避免超过 40 字的长句，分割或重写 |
| 专业词汇 | 首次出现附简短解释 |
| 白话程度 | 避免过度书面化（"然而"、"此乃"等），保持亲切感 |
| 数字格式 | 阿拉伯数字（3个月，而非三个月） |

---

## 5. 标题吸引力评分

对 5 个候选标题，各给出 1-10 分并简短说明：

- **点击欲**：看到标题是否想点开
- **准确性**：标题是否准确反映内容（无标题党）
- **关键词**：是否包含读者可能搜索的词

推荐排序（从高到低），但最终由用户选择。

---

## 审核摘要格式

输出一段话（100字以内）：
```
审核完成。[X个问题已自动修复：简述]。[如有保真度疑问，列出]。
标题推荐顺序：[1/3/2/5/4]。文章整体评分：[X/30]。
```
